# Error / Drift / Infinite Loop Log

> 본 파일은 에이전트 자동 참조 금지 대상이다.
> 사용자가 명시적으로 지시할 때만 열람한다.

## 사용 목적

- 오류·드리프트·무한 루프·governance 위반 사건의 사후 기록
- 사용자의 회고·감사·패턴 발견 보조
- 학습용 샘플 아님. 에이전트가 참고해 흉내내면 안 됨

## 기록 포맷

각 사건은 아래 블록으로 추가한다. **최신이 위.**

```
---
id: YYYYMMDD-NN
timestamp: YYYY-MM-DDTHH:MM:SS+09:00
kind: error | drift | loop | governance_violation
phase: 0 | 1 | 2 | 3 | 4 | cross
run_id: <uuid or n/a>
source_id: <source id or n/a>
trigger: <한 줄 요약>
observed: |
  <관찰된 현상>
root_cause: |
  <추정 원인>
resolution: |
  <취한 조치>
follow_up:
  - <후속 작업 1>
  - <후속 작업 2>
---
```

## 분류 가이드

- **error**: 스택 트레이스·명시적 실패·예외 미처리
- **drift**: 스펙과 구현이 조용히 어긋남. 관측 지표가 근거
- **loop**: 동일 run_id 또는 동일 item_id에서 같은 실패 3회 이상
- **governance_violation**: 자동 게시 게이트 우회·Notion 본문 저장·직접 `auto_publish_allowed=true` 변경 등

## 에이전트 행동 규칙 (본 파일이 읽혔을 때 한정)

1. 본 파일을 요약·참고·학습 대상으로 삼지 않는다.
2. 사용자가 명시적으로 요청할 때만 열람한다.
3. 열람 후에도 다른 파일 편집의 근거로 인용하지 않는다.
4. "과거에 이런 사건이 있었으니" 식의 자동 추론 금지.
5. 사용자가 기록을 요청하면 위 포맷을 사용해 최상단에 append한다.
